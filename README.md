# Margi_-settlement

名大SKYの月次マージン清算書を自動生成するツール。

対象サービス: プログラミング / 将棋 / 文理ヴィクトリー / 速読

## 使い方

共通の引数で、`service` を切り替えるだけで 4 サービスに対応する。

### プログラミング清算書の生成

```bash
python margin_settlement.py programming \
  --source  "Y:\_★20170701作業用\【エデュプラス請求書】\【業者請求書】エクセルbackup\2026年3月21日送信分\2026.3.23送信.xlsm" \
  --template "Y:\_★20170701作業用\【エデュプラス請求書】\プログラミング清算書\プログラミング売上管理表_202603月分.xlsx" \
  --output   "Y:\_★20170701作業用\【エデュプラス請求書】\プログラミング清算書\プログラミング売上管理表_202604月分.xlsx" \
  --month    2026-04
```

### 対応表（source → settlement）

source .xlsm は **1ヶ月前の送信分** を使う。

| 清算書 | source folder |
|---|---|
| 202603月分（3月分） | 2026年2月17日送信分 |
| 202604月分（4月分） | 2026年3月21日送信分 |
| 202605月分（5月分） | 2026年4月18日送信分 |

### 引数

| 引数 | 説明 |
|---|---|
| `service`    | `programming` / `shogi` / `bunri` / `sokudoku` |
| `--source`   | 当月の業者請求書 `.xlsm` |
| `--template` | 前月の清算書 `.xlsx`（雛形として使用） |
| `--output`   | 出力先のパス |
| `--month`    | 対象月 `YYYY-MM` |
| `--skip-google-sheet` | 新規塾検出時の Google Sheet 照会をスキップ |
| `--add-shops` | カンマ区切りの家族IDを `報告書` B列に追加 |
| `--sales-master` | `プログラミング営業管理` 用 CSV URL/ローカルパス（programming のみ）|

## 自動処理内容

全サービス共通:
1. 前月清算書をコピーして新ファイル作成
2. 各カテゴリーシートを source から上書き（集約列 T/U, O/P, U/V などへの家族ID重複集約を含む）
3. `報告書!D1` に対象月を設定
4. 新規家族ID候補を検出（報告書に未登録の家族IDをリスト表示）

### サービス別の貼り替えシート

**プログラミング** (`config/programming.yaml`)
- `らくらく　ユーザー基本情報貼り付ける`
- `保護者情報DL貼付⑩AKへ`
- `④_2プロ_管理者ＩＤ` ← source `④プロ_管理者ＩＤ`
- `④_3プロ_生徒ＩＤ` ← source `④プロ_生徒ＩＤ`
- `④ゲームクリエイター生徒ID`
- `④_4カルチャ加盟金`
- `④カルチャー_基本料金`

**将棋** (`config/shogi.yaml`)
- `保護者情報DL貼付⑩AKへ`
- `④_4スマイル将棋_生徒ID` ← source `④スマイル将棋_生徒ID`
- `④スマイル将棋_入会金`
- `④スマイル将棋管理者ＩＤ`

**文理ヴィクトリー** (`config/bunri.yaml`)
- `保護者情報DL貼付⑩AKへ`
- `文理ビクトリー_基本料金`（src/dst 同名、ヴ無し）
- `文理ヴィクトリー初期費用` ← source `文理ビクトリー初期費用`
- `文理_生徒` ← source `④文理ビクトリー利用料`
- `文理V_オンライン個別` ← source `④文理ビクトリーチケット`
- `文理ヴィクトリー有料講座` ← source `④文理ビクトリー有料講座`
- `文理おまかせコース` ← source `④文理おまかせコース`

**速読** (`config/sokudoku.yaml`)
- `保護者情報DL貼付⑩AKへ`
- `速読ID利用料`（F列の項目テキストから生徒数を MID 抽出して R 列に書き込む）

## 手動作業として残る項目

- `報告書!AJ3`（プログラミング: 名大SKY直営分マイクラID数）は手入力
- 各サービスの `報告書!L6`（将棋: 名大SKY直営分の仕入れ額）等、家族ID=999999 行は手入力
- 新規塾の `報告書!B列` への追加は、現状は候補を表示するだけで手動追加が必要
- 文理ヴィクトリー/速読の `保護者情報DL貼付⑩AKへ` は source からそのままコピーするが、既存の清算書は手動で行を絞り込んでいる場合があるため、必要に応じて調整

## セットアップ

```bash
pip install -r requirements.txt
```

### Google Sheets API認証（新規塾自動追加を使う場合）

サービスアカウント JSON を `credentials.json` として配置するか、環境変数 `GOOGLE_APPLICATION_CREDENTIALS` でパス指定。

## 検証

各サービスに対して、前月テンプレから既存の清算書を再生成し、セル単位で diff する検証スクリプトを用意している。

```bash
# プログラミング 3月分を再現
python scripts/verify_march.py

# 将棋 3月分を再現
python scripts/verify_shogi.py

# 文理ヴィクトリー 3月分を再現
python scripts/verify_bunri.py

# 速読 9月分(2025)を再現
python scripts/verify_sokudoku.py
```

検証上の注意:
- `報告書` の diff は **手入力セル**（名大SKY直営分の額、手動追加した新規塾行など）に限定されているのが期待値
- `保護者情報DL貼付⑩AKへ` の大量 diff は、既存清算書が手動で行を削除している場合の整列ズレ
- 各カテゴリーシートの formula vs 値の diff は本ツールが `data_only=True` で source の関数を解決するための差分（機能的には同等）
